# Рекомендуемые параметры для запуска RL Trading Pipeline

## Быстрый запуск (для тестирования и отладки)

Для быстрой проверки работоспособности системы рекомендуется использовать следующие параметры:

```bash
python run_pipeline.py --data-days 90 --timesteps 50000 --max-iterations 3 --min-return 0.01 --min-sharpe 0.3 --max-drawdown 0.2
```

**Пояснение:**
- `--data-days 90`: Используется 90 дней исторических данных (быстрее подготовка данных)
- `--timesteps 50000`: Меньше шагов обучения (быстрее обучение)
- `--max-iterations 3`: Максимум 3 итерации (быстрее завершение)
- `--min-return 0.01`: Только 1% минимальная доходность (легче достичь)
- `--min-sharpe 0.3`: Более низкий порог Шарпа (легче достичь)
- `--max-drawdown 0.2`: 20% максимальная просадка (более терпимо)

## Рекомендуемые параметры для полноценного запуска

Для получения качественной модели с хорошими показателями прибыльности используйте следующие параметры:

```bash
python run_pipeline.py --data-days 365 --timesteps 200000 --max-iterations 5 --min-return 0.02 --min-sharpe 0.5 --max-drawdown 0.15
```

**Пояснение:**
- `--data-days 365`: Используется год исторических данных для полноценного обучения
- `--timesteps 200000`: Достаточное количество шагов для качественного обучения
- `--max-iterations 5`: 5 итераций для поиска оптимальных параметров
- `--min-return 0.02`: 2% минимальная доходность (реалистичная цель)
- `--min-sharpe 0.5`: Коэффициент Шарпа 0.5 (приемлемый риск/доходность)
- `--max-drawdown 0.15`: 15% максимальная просадка (разумный риск-контроль)

## Амбициозный запуск (для получения высококачественной модели)

Для поиска действительно высококачественной модели с отличными показателями:

```bash
python run_pipeline.py --data-days 365 --timesteps 500000 --max-iterations 8 --min-return 0.03 --min-sharpe 0.7 --max-drawdown 0.1
```

**Пояснение:**
- `--data-days 365`: Полный год исторических данных
- `--timesteps 500000`: Большое количество шагов для тщательного обучения
- `--max-iterations 8`: Много итераций для поиска оптимальных параметров
- `--min-return 0.03`: 3% минимальная доходность (высокая цель)
- `--min-sharpe 0.7`: Высокий коэффициент Шарпа (качественная модель)
- `--max-drawdown 0.1`: Только 10% максимальная просадка (консервативный риск)

## Запуск без установки зависимостей

Если зависимости уже установлены, можно пропустить установку:

```bash
python run_pipeline.py --skip-install --data-days 365 --timesteps 200000 --max-iterations 5 --min-return 0.02 --min-sharpe 0.5 --max-drawdown 0.15
```

## Рекомендации по выбору параметров

1. **Для первого запуска**: Используйте параметры быстрого запуска, чтобы убедиться, что система работает правильно.

2. **Для реальной модели**: Используйте рекомендуемые параметры для полноценного запуска - это оптимальный баланс между качеством модели и временем обучения.

3. **Для высококачественной модели**: Используйте амбициозные параметры, если у вас есть достаточно времени и вычислительных ресурсов.

4. **Для ограниченных ресурсов**: Уменьшите `--timesteps` и `--max-iterations`, но возможно потребуется снизить требования к `--min-return`, `--min-sharpe` и `--max-drawdown`.

## Важные замечания

- Обучение может занять несколько часов в зависимости от параметров
- Рекомендуется запускать на системе с GPU для ускорения обучения (но для CPU также доступна установка PyTorch)
- Убедитесь, что у вас достаточно свободного места на диске для сохранения моделей и логов
- Мониторьте использование памяти во время обучения
- Скрипт `run_pipeline.py` автоматически устанавливает все необходимые зависимости, включая PyTorch и Stable-Baselines3
